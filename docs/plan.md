# AIGC 营销内容生成平台 — 总体方案（v3）

> 本文档是产品与技术方案的**决策存档**。具体目录结构与修改入口见 [`../AI_GUIDE.md`](../AI_GUIDE.md)。

---

## 一、项目定位

- 服务对象：个人创作者 / 3–10 人小型市场或运营团队
- 部署形态：**本地开发优先**（Docker Compose），后续可迁移到云服务器或 Vercel + 托管服务
- 核心价值：用一个统一界面 + 品牌知识库，把多个大模型/图模型/视频模型串成一条营销内容流水线
- 不做：多租户、SSO、复杂审批流、计费系统、企业 SLA

---

## 二、核心功能模块

1. **广告文案生成（单条精修）** — 多平台、RAG 注入品牌上下文
2. **社媒批量好评文案** — 50–100 条差异化、避免雷同、可导出
3. **营销活动策划**（P5，多步 Agent）
4. **海报生成** — 3 步交互式：上传素材 → 输入风格/重点 → 可编辑画布预览
5. **产品宣传图生成**
6. **图文短视频（MVP）** — 图片 + TTS + FFmpeg
7. **AI 视频生成（Wan 2.5，P5）**
8. **品牌知识库（RAG）**
9. **历史素材管理 & 收藏**

---

## 三、系统架构

```
[Next.js 前端 / Polotno 画布] ──► [FastAPI API] ──► [Postgres + Redis + Qdrant + MinIO]
                                       │
                                       ├─► [Celery Worker] ──► [模型路由层]
                                       │                          ├─ DeepSeek-V4（文案/策划初稿）
                                       │                          ├─ Qwen3.7-Max（文案/策划润色）
                                       │                          ├─ Qwen-Image-2.0 / Qwen-VL（DashScope）
                                       │                          ├─ DeepSeek-V4（视频脚本）
                                       │                          ├─ Wan2.7-Video（视频片段）
                                       │                          └─ Edge-TTS / CosyVoice
                                       └─► [内容安全 API]
```

部署 3 选 1：
- **A**：本地 + Cloudflare Tunnel + Vercel 前端（个人，基础设施 ≈¥0/月）
- **B**：Vercel + Inngest/Trigger.dev + Neon + Upstash + Modal/Replicate + R2（小团队零运维）
- **C**：Vercel 前端 + 1 台 4C8G 云服务器跑后端（小团队最省心，¥150–300/月）

**当前采用：本地开发，暂不部署。**

## 五、规则审核（4 类）

1. **敏感词审核**：涉政/涉黄/涉暴 + 《广告法》违禁词（最佳/第一/国家级/绝对/100%）+ 行业禁用词。DFA 实现。
2. **品牌合规**：品牌名拼写、必含 Slogan/®/™、禁用竞品名。正则 + 黑白名单。
3. **格式规范**：长度限制（小红书 ≤1000、X ≤280、公众号标题 ≤64）、必含 CTA/Hashtag、Emoji 上限。
4. **视觉合规（海报）**：Logo 检测、品牌色 ΔE 色差。

兜底：Qwen2.5-7B 短 prompt 语义判定（约 ¥0.001/次）。

## 七、Agent 使用边界

仅 **P5** 阶段，且仅用于多步推理 + 工具调用场景：
- 营销活动策划（目标拆解 → RAG → DeepSeek-V4 策略框架 → Qwen3.7-Max 润色大纲 → 自审改写）

P1–P4 全部直接函数调用，不引入 LangGraph。

## 八、广告模块（2 个子模块）

### 8.1 单条文案精修
输入：产品名、卖点、人群、平台、风格、长度
输出：N 个标题 + 1 段正文 + Hashtag + Emoji

**两步生成流程**：
1. **DeepSeek-V4** 根据产品信息生成初版框架（强调数据驱动、卖点结构清晰）
2. **Qwen3.7-Max** 接收初版，输出润色版本（专业表达、符合中文营销习惯）

### 8.2 批量好评（防雷同）
- LLM 每次出 5–10 条，多轮调用
- 人设池随机抽取（宝妈/学生/上班族/数码爱好者…）
- 句式模板随机化
- 后处理去重：编辑距离 + 向量相似度 > 0.85 重写
- 输出 CSV
- UI 显著提示：仅供内部参考，禁用于刷单等违法用途

## 九、海报（3 步交互式）

**Step 1 素材上传**：产品图（必填）+ Logo（可选）+ 参考海报（可选）
**Step 2 输入**：风格 + 重点（主/副标题、卖点、CTA）+ 画幅 + 配色
**Step 3 生成 + 可编辑**：
1. Qwen-VL 提取产品图视觉特征
2. LLM 生成 image_prompt
3. Qwen-Image-2.0 出底图（不含文字）
4. 模板引擎初步合成
5. 前端 **Polotno SDK** 可拖拽/改字/换图/重生成底图
6. 后端 Pillow 导出 PNG/JPG

## 十、AI 视频生成（3 步交互式流水线）

> 每步完成后**暂停等待用户确认**，用户可修改内容后再继续下一步。

```
[STEP 1] DeepSeek-V4 生成脚本 + 分镜描述
         输入：产品名、卖点、风格、时长、镜头数
         输出：脚本 JSON（含旁白、每镜头 scene_desc）
         ↓ [用户可编辑每个 scene_desc，确认后继续]

[STEP 2] Qwen-Image-2.0 逐镜头生成参考图
         输入：scene_desc（每镜头）
         输出：每镜头一张图（存 MinIO）
         ↓ [用户可替换不满意的镜头图，确认后继续]

[STEP 3] Wan2.7-Video 逐镜头生成视频片段
         输入：每镜头参考图 + scene_desc
         输出：每镜头一段 MP4 片段（存 MinIO）
         ↓ [自动 FFmpeg 拼接 + 字幕 + BGM → 最终 MP4]
```

**交互节点设计**：
- 后端每步完成后更新 `video_tasks.stage`（`script_done` / `images_done` / `clips_done`）
- 前端轮询 stage 变化，弹出确认卡片（可编辑 JSON / 可重新生成单张图 / 可重新生成单个片段）
- 用户点击"确认进入下一步"后，前端 POST `/api/video/confirm` 触发下一阶段 Celery 任务
- 任意步骤支持**局部重试**：单镜头图或单镜头视频可独立重新生成

## 十三、近期目标任务（落地优先序）

### T1. 文案策划可直接交付（最高优先）

- 稳定 `/api/copy/generate` 的成功率与错误提示
- 完善提示词模板与参数边界
- 输出固定验收样例（可复测）

### T2. 批量好评可直接交付

- 稳定 `/api/reviews/generate` 质量（去重、风格、多人设）
- CSV 导出体验与字段约定固化
- 目标条数 20–100 的性能与可用性验证

### T3. AIGC 图片（海报）可直接交付

- 稳定 `/api/poster/generate` + 轮询链路
- 强化 Celery 任务可观测性与失败重试
- 图片 URL 可访问性与下载体验验证

### T4. 测试与交付门槛（当前必须通过）

- 后端聚焦测试：copy/reviews/poster 全绿
- 前端构建通过（含 `/copy` `/reviews` `/poster` 页面）
- 形成一键复测脚本与验收 checklist

### T5. 视频与 Agent 后续增强（延后）

- 营销活动多步 Agent 策划（P5）
- Qdrant 向量检索、Wan 视频模型、音频字幕高级能力
- 由后续需求触发再进入执行序列
